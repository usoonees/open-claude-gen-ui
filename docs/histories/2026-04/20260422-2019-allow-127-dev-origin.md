### User Query

> `pnpm dev` starts successfully, but opening `http://127.0.0.1:3000/` fails with a Next.js dev-origin warning. Why can that URL not be opened?

### Changes Overview

- Area: local development configuration.
- Key actions:
  - Added `allowedDevOrigins: ["127.0.0.1"]` to `next.config.ts` so the Next.js 16 dev server accepts HMR and related dev resources when the app is opened via `http://127.0.0.1:3000`.
  - Updated the frontend guide to document that both `localhost` and `127.0.0.1` are valid local dev entry points after the config change.

### Design Intent

Next.js 16 blocks cross-origin access to dev-only resources by default. Opening the app on `127.0.0.1` while the dev server advertises `localhost` caused the browser to request dev assets from a different origin, so HMR failed and the page could not load cleanly. This change keeps the fix narrow by allowing only the loopback host that the user actually tried to use.
