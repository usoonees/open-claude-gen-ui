### Changes Overview

- Area: chat persistence and sidebar history titles
- Key actions:
  - Added a server-side chat title generator that uses the existing Volcengine model to create short sidebar titles from the opening conversation context.
  - Changed the save flow so a new sidebar item first appears with the raw first prompt, then resolves the generated title in the background instead of blocking the initial save.
  - Added persisted title state so the client can poll only while a generated title is pending and stop once it is resolved or falls back.
  - Added a sidebar title refresh animation so generated titles replace placeholder prompt text smoothly instead of snapping.
  - Tightened optimistic save handling so failed history saves surface back to the local sidebar fallback path instead of silently succeeding.
  - Updated architecture, frontend verification guidance, and release notes for the new title behavior.

### Design Intent

Keep the chat creation path immediate and legible by showing the user's real prompt first, then upgrade the sidebar label once the model finishes generating a shorter title. Persisting the pending/ready title state keeps reloads and later saves consistent, while the old first-prompt truncation remains as the no-inference fallback.
