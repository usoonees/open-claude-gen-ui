### What Changed

- Area: chat frontend message labeling and streamed message metadata.
- Key actions:
  - Added typed chat message metadata so assistant UI messages can carry the exact `modelId` that generated that turn.
  - Stamped the active request model onto assistant messages at stream time so the label survives live rendering, reloads, and saved-chat history.
  - Updated the assistant message label rendering to show `Assistant(model_name)` when model metadata is present, while keeping a plain `Assistant` fallback for older messages.
  - Extended frontend verification guidance to cover per-turn model labels and persisted assistant message metadata.

### Why

The active picker model is mutable across a conversation, so a static `Assistant` label loses important provenance once the user switches models mid-chat. Persisting the model id on each assistant turn makes the UI legible after model changes and keeps saved chats analyzable without inferring which model produced which response.
